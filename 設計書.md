設計書：GitHub Dynamic Greeting Badge

GitHubのREADMEに埋め込み可能な、時間帯によってメッセージが変化する動的画像生成システム。

1. システム概要

ユーザーがGitHubプロフィール（README.md）を表示した際、閲覧者の時間帯（JST）に合わせた挨拶と「気が利いたメッセージ」を画像として生成・表示する。

2. 技術スタック

Framework: Next.js (App Router)

Image Generation: next/og (Satoriベース)

Deployment: Cloudflare Pages (または Vercel)

Language: TypeScript

3. 主要機能

① 時間判定ロジック

サーバーサイド（Edge Runtime）で現在時刻を取得し、以下の3区分に分類する。

朝 (05:00 - 10:59): 「おはようございます」系

昼 (11:00 - 17:59): 「こんにちは」系

夜 (18:00 - 04:59): 「こんばんは」系

② メッセージ選択機能

各時間帯に応じたメッセージを複数用意し、ランダムまたは日付に基づいて1つ選択する。

例 (昼): 「進捗はどうですか？コーヒーを淹れて一息つきましょう。」

例 (夜): 「今日もお疲れ様です。暗い画面の見すぎに注意してくださいね。」

③ 画像生成 (SVG/PNG)

next/og を使用し、Tailwind CSS風のスタイルでデザインされた画像を動的に生成する。

サイズ: 600px × 160px (READMEで収まりが良いサイズ)

デザイン要素: アイコン、挨拶タイトル、一言メッセージ、現在の時刻（秒なし）。

4. API 仕様

エンドポイント

GET /api/badge

レスポンスヘッダー (重要)

GitHubのキャッシュ回避（Camo対策）のため、以下のヘッダーを付与する。

Cache-Control: no-cache, no-store, must-revalidate

Pragma: no-cache

Expires: 0

5. READMEへの埋め込み方法

![Greeting Badge](https://your-app-domain.com/api/badge?t={TIMESTAMP})


※ ?t= を付けることで、ブラウザキャッシュをより強力に回避できる。

6. 拡張アイデア

GitHub Actions連携: 自分が最後にコミットした時間を表示する。

ランダム名言: 技術者の名言を100個くらいリスト化して表示する。

季節対応: お正月、クリスマス、誕生日などの特殊メッセージ。
